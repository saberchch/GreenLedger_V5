02 Data Normalization & Mapping

Before calculation, software must accept dirty, unstructured data (like an AP spend report) and structure it. Often, NLP (Natural Language Processing) is used to map chaotic string descriptions to strict database factor IDs.

def map_to_factor(description_string, context_tags):

    vector = nlp_model.encode(description_string)

    best_match = factor_db.query(vector, top_k=1)

    return best_match.ef_id

04 The Immutable Ledger

Once emissions are calculated, they must be protected from tampering. GreenLedger utilizes a role-based approval flow culminating in an immutable audit hash.

1. Worker Input

A facility manager inputs "7,000 kWh" and uploads the utility PDF.

2. Admin Approval

The Org Admin compares the input to the PDF evidence and clicks "Approve".

05 Build vs. Buy

Why pay for a SaaS platform instead of building internal Excel macros? Carbon accounting is deeply complex and constantly shifting.

Feature Segment Spreadsheets (In-House) Carbon SaaS (GreenLedger)
Factor Database Must manually download, update, and fix VLOOKUPs yearly. Auto-updated via API integrations to ADEME/DEFRA.
Auditability No version control. Anyone can overwrite a cell. Role-based access, document vaults, hash locking.
Standards Logic Must read 200-page GHG Protocol PDFs to design math. Accounting logic is hardcoded by platform experts.
